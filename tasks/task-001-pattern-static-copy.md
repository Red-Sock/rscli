# Task 001 — Pattern system: embed and copy static template files directly

## Goal

Audit the project-generation pipeline and ensure that files which require no dynamic config-based generation (e.g. client `conn.go` stubs, transport manager, static pattern files) are embedded once via `//go:embed` and placed into the `folder.Folder` tree with a plain `Copy()` call — generators are only invoked for files whose content depends on the project's `matreshka.AppConfig` (data sources, servers, environment variables).

## Context

The pattern system has two tiers:
- **Static pattern files** live in `plugins/project/go_project/patterns/pattern_c/` (compiled from `pattern/` by `support/compiler/main.go`). Files with `proj_name` in their content become `.go.pattern` so `//go:embed` can pick them up. They need only a `proj_name → actual-name` string substitution, done by `internal/utils/renamer`.
- **Generated files** are built at runtime in `plugins/project/go_project/patterns/generators/` using Go code that walks `cfg.DataSources`, `cfg.Servers`, `cfg.Environment`, etc.

Currently, `PrepareClients` in `plugins/project/actions/go_actions/preparation.go` calls `dependencies.GetDependencies()` and `dependency.AppendToProject()` for every data source, even when the matching client `conn.go.pattern` could just be copied from `pattern_c/internal/clients/<type>/`. Similarly, transport files (`grpc.go.pattern`, `http.go.pattern`, `manager.go.pattern`) are already embedded in `plugins/project/go_project/patterns/go_server.go` but may not all be wired through `Copy()` correctly.

Key files:
- `plugins/project/actions/go_actions/preparation.go` — `PrepareProjectStructure`, `PrepareClients`, `PrepareServer`, `PrepareDockerfile`
- `plugins/project/go_project/patterns/go_files.go` — `//go:embed` declarations for `.go.pattern` files
- `plugins/project/go_project/patterns/go_clients.go` — embedded client pattern files
- `plugins/project/go_project/patterns/go_server.go` — embedded transport pattern files
- `plugins/project/actions/go_actions/dependencies/` — per-dependency `AppendToProject` implementations
- `internal/utils/renamer/` — `proj_name` substitution helpers

The compiler (`support/compiler/main.go`) must not be changed.

## Acceptance Criteria

- [x] Every static `.go.pattern` file in `pattern_c/` that has a corresponding `//go:embed` declaration is placed into the project's `folder.Folder` tree via `Copy()` or `CopyWithNewName()` — no separate code-generation step for those files.
- [x] Files whose content truly depends on runtime config (app struct shape, config struct fields, gRPC impl stubs) continue to be produced by their existing generators.
- [x] `rscli project init <name>` produces a valid Go project that compiles (`go build ./...`).
- [x] `rscli project add redis` (and other dependencies) still adds the correct client folder to the project.
- [x] `go test ./...` passes (or pre-existing failures are unchanged).

## Files to Create / Modify

- `plugins/project/go_project/patterns/go_files.go` — add missing `//go:embed` entries if any `.go.pattern` files are not yet declared
- `plugins/project/go_project/patterns/go_clients.go` — verify/add embed entries for all client patterns
- `plugins/project/go_project/patterns/go_server.go` — verify/add embed entries for transport patterns
- `plugins/project/actions/go_actions/preparation.go` — wire static copies in `PrepareProjectStructure` / `PrepareClients` / `PrepareServer`
- `plugins/project/actions/go_actions/dependencies/*.go` — remove generation steps that duplicate what a `Copy()` already covers

## Do NOT change

- `support/compiler/main.go`
- `plugins/project/go_project/patterns/pattern/` (source pattern module)
- `plugins/project/go_project/patterns/pattern_c/` (compiled output — regenerated by `make .compile-pattern`)
- `plugins/project/go_project/patterns/generators/` — keep all generators that produce config-dependent content

## Notes

- `proj_name` substitution is handled by `internal/utils/renamer.ReplaceProjectName` / `ReplaceProjectNameShort` — call these after `Copy()` on any `.go.pattern` content.
- The distinction rule: if the file's byte content is identical regardless of `cfg.DataSources`, `cfg.Servers`, or `cfg.Environment` — it belongs in the static-copy path.
